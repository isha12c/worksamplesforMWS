darkroom

A digital gallery for browsing 35mm film photography, roll by roll. Pick a camera, develop the roll, and step through frames the way you'd sort through a contact sheet — built to feel closer to handling physical film than scrolling a camera-roll app.

View the live gallery ↗

Build issues — frame log

FRAME 01

Rotation

Scanned film carries far less embedded metadata than a phone photo — no reliable orientation flag to read. Scans came in rotated inconsistently, so I wrote scripts to batch-correct and normalize orientation before anything could go into the gallery.

FRAME 02

File legibility

Using Claude to help process the scans directly, file naming and format inconsistencies from scanning meant it couldn't reliably read every file. I ended up building workarounds to standardize files on ingestion instead of fixing it per-file.

FRAME 03

What's next in progress

Right now, getting a roll onto the site is a manual process. I'm building a companion app and backend so I can upload directly from my phone, skipping the scan-and-rotate workflow entirely.

Build issues

Infrastructure shift

An AWS change broke deployment mid-flight — the app started throwing read errors and returning inaccurate, sometimes negative, scores until I traced it back.

Silent failure handling

Longer policy pages caused the scoring Lambda to time out. Instead of failing cleanly, it returned negative scores — I had to rework the function to fail visibly rather than return bad data.

Dependency debt

Audited and updated outdated packages across the codebase to close security gaps and reduce version-compatibility risk.

Vendor lock-in

After the AWS incident, I tested Google Cloud as a working backup so a single provider outage can't take the whole tool down again.
